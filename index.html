Project Introduction
WHAT WE BUILT

Robotic grasping is a fundamental task in manipulation and automation. Conventional systems rely on manually tuned heuristics — they break under object variability and have no mechanism to detect grasp failures before the task is already ruined.

This project designed a vision-driven deep learning pipeline for a 4-DOF robotic arm. The core idea: score candidate grasps using a trained model before executing, then verify the pick after lifting. Only confirmed picks proceed to color-based sorting into designated bins.

The hardware pipeline was designed, components were acquired, and partial integration was carried out — the arm, servos, gripper, camera, and Arduino were all assembled. Full end-to-end hardware operation was not achieved due to integration challenges.

To validate the complete pipeline logic, the entire system was implemented in PyBullet simulation using the Franka Panda robot — IK trajectory control, pick verification via contact points, and color sorting all run correctly in simulation.

Machine Learning
Deep Learning Grasp Prediction
A trained model was designed to predict grasp quality scores for candidate regions and select the best pose before execution.

4-DOF Motion + Kinematics

Automation

Problem Statement
WHY THIS MATTERS
01
Heuristic Fragility
Conventional grasping methods relied on heuristics or manual tuning — these failed under object variability in position, orientation, or lighting.
02
Late Failure Detection
Grasp failures were detected only after task failure — the arm proceeded to the drop zone even with nothing in its gripper, wasting entire cycles.
03
No Post-Grasp Check
Without visual pick verification after lifting, there was no way to confirm the object was actually grasped. System reliability was fundamentally limited.
04
No Pre-Execution Scoring
Without a learned quality estimator, all candidate grasps looked identical. The arm executed the first feasible pose regardless of its actual quality.
05
Fixed Sorting Rules
Rule-based sorting broke under variability. A vision-based approach reading color from each verified pick was far more robust.
→
Our Approach
Score grasps before execution using deep learning. Verify picks after lifting via contact points. Sort by color. Validate fully in simulation.

Simulation Implementation
WHAT WAS ACTUALLY BUILT

The entire pipeline was implemented in PyBullet simulation using the Franka Panda robot model. This validated all core logic — IK-based motion, contact-point pick verification, and color-based sorting across 30 randomly placed cubes.

simulation.py — Franka Panda · PyBullet · 240 Hz
# ===== CONFIG ===== SIM_HZ = 240 # physics rate NUM_CUBES = 30 # random on table ARM_FORCE = 2000 GRIP_FORCE = 600 LIFT_HEIGHT = 0.9 # m APPROACH = 0.12 GRASP_OFFSET = 0.01   # Only RED, GREEN, BLUE sorted TARGET_COLORS = ["RED","GREEN","BLUE"]   DROP_ZONES = { "RED": [0.40, 0.60, 0.63], "GREEN": [1.00, 0.60, 0.63], "BLUE": [0.70, -0.60, 0.63] }   # ===== MAIN LOOP ===== for cube in cubes: if cube["name"] not in TARGET_COLORS: continue # skip YELLOW/PURPLE/CYAN   agent.open_gripper() agent.move_to([x, y, LIFT_HEIGHT]) agent.move_to([x, y, z+APPROACH]) agent.move_to([x, y, z+GRASP_OFFSET]) agent.close_gripper() pick_attempts += 1   # ← PICK VERIFICATION if not agent.verify_grasp(cube_id): continue # failed — skip sort grasp_success += 1   constraint = p.createConstraint( robot, 11, cube_id, -1, p.JOINT_FIXED, ...) agent.move_to(drop) agent.open_gripper() p.removeConstraint(constraint) place_success += 1   ============================== SIMULATION RESULTS ============================== Total Cubes: 30 Target Cubes: ~17 (R+G+B) Grasp Success Rate: 94.1 % Sorting Accuracy: 100.00 %

Simulation — Core Feature
Pick Verification
CONTACT POINT VERIFICATION

Pick verification was the key feature that prevented empty-gripper sort trips. In simulation, PyBullet's physics contact query served as the ground truth check.

01
Close Gripper + Lift
Gripper closed with GRIP_FORCE = 600. Arm lifted to LIFT_HEIGHT = 0.9 m before verification ran. Gravity had fully acted on the grasp at this height.
02
Contact Point Query
p.getContactPoints(robot, cube) queried PyBullet's physics engine for active contacts between any robot link and the cube body. Returns a list of contact structures.
03
Evaluate Contact Count
If len(contacts) > 0 — pick was verified. The gripper was holding the cube and sorting proceeded. If 0 contacts — the pick had failed.
04
Reject Failed Pick
On failure, continue skipped MOVE_TO_BIN and RELEASE entirely. The arm made no wasted trip to the drop zone. The next cube was processed immediately.
05
Attach Constraint + Sort
On success, a JOINT_FIXED constraint was created between robot link 11 (EE) and the cube — guaranteeing the cube stayed in the gripper during transport. Removed after drop.
verify_grasp() — simulation.py
def verify_grasp(self, cube): contacts = p.getContactPoints( self.robot, cube ) return len(contacts) > 0   # In sorting loop: agent.close_gripper() pick_attempts += 1   if not agent.verify_grasp(cube_id): continue # reject   grasp_success += 1   constraint = p.createConstraint( robot, 11, cube_id, -1, p.JOINT_FIXED, [0,0,0],[0,0,0],[0,0,0] )
Hardware Equivalent
On the physical arm, verify_grasp() was designed to be a camera-based visual presence check — a post-lift image was to be captured and analyzed to confirm the object in the gripper before the arm moved to the drop zone. Same logic, different sensor.
Simulation — Color Sorting
Color-Based Sorting
THREE BINS, ZERO MISTAKES

30 cubes were spawned randomly across the table in 6 colors. Only RED, GREEN, and BLUE were picked and sorted. Each had a fixed drop zone coordinate. 100% sorting accuracy was achieved on all verified picks.

RED
[ 0.40, +0.60, 0.63 ]
Near-left drop zone. Arm approached from LIFT_HEIGHT, lowered to z = 0.65, opened gripper, constraint removed after release.
GREEN
[ 1.00, +0.60, 0.63 ]
Far-left zone — maximum x-reach. Tested the full extension limit of the IK solver at the workspace boundary.
BLUE
[ 0.70, -0.60, 0.63 ]
Opposite side — negative Y. The arm swung across the full workspace width, testing joint limit handling during cross-table motion.
